# 出现的问题与解决方案 (2026-04-06 更新)

## 1. 语义切片带来的断层
- **问题**：固定长度 (Fixed-Length) 切片破坏了工业手册中的段落与表格，导致 Embedding 后的向量检索出现“牛头不对马嘴”。
- **解决方案**：引入基于特定符号与段落感知的切片（RecursiveCharacterTextSplitter），优先保证段落的完整性，并提取元数据（如章节信息）。

## 2. 混合检索排名分布差异
- **问题**：Elasticsearch (BM25) 与 Qdrant (Distance) 生成的初始分数分布不一，无法直接加权。
- **解决方案**：采用 RRF (Reciprocal Rank Fusion) 算法，通过排位倒数求和，规避了分值归一化难题。

## 3. 工业设备型号匹配困难
- **问题**：纯向量模型对型号敏感度极低，搜索“VFD-C2000”常返回其他变频器，导致检索召回质量差。
- **解决方案**：在检索前应用同义词词库（Synonyms），并通过 Elasticsearch 的 BM25 补偿关键词得分，实现精确定位。

## 4. 交错编码器 (Cross-Encoder) 性能瓶颈
- **问题**：由于交叉编码模型需要计算每一对 Query-Doc 的语义关联，速度远慢于双编码器，在大规模文档下响应迟钝。
- **解决方案**：采用“粗排 + 精排”两阶段策略。粗排（BM25 + Vector）召回 Top-50，模型仅对 Top-12 进行精排计算。

## 5. 本地微调显存限制
- **问题**：Cross-Encoder 模型进行微调时，本地显存不足 (OOM)，导致模型难以充分微调以适应特定工业场景。
- **解决方案**：计划将训练同步到 Google Colab (GPU-T4) 进行，并探究使用 LoRA 等轻量化微调策略。
